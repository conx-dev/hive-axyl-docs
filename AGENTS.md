# Hive Axyl 외부 문서 작성 지침

## 문서 목적

이 저장소는 외부 게임 개발자가 Hive Axyl SDK와 Console 웹 UI를 설치하고 사용하는 방법을 설명합니다.

## 작성 범위

- Web, Android, iOS, Unity, Godot SDK의 설치, 초기화, 공개 API 사용법을 작성합니다.
- Console 웹 UI에서 사용자가 확인할 수 있는 화면, 입력값, 작업 순서, 결과를 작성합니다.
- 외부 개발자에게 공개된 API 계약, 설정값, 오류 코드, 복구 방법을 작성합니다.
- 로그인, 권한, 초대와 관련된 내용은 사용자가 제품을 이용하는 데 필요한 화면 동작과 결과만 작성합니다.

## 작성 금지

- 내부 서버 구성, 서비스 이름, 서비스 간 호출 관계, 네트워크 토폴로지를 작성하지 않습니다.
- 내부 호스트, IP 주소, 포트, 배포 환경, 인프라 구성을 작성하지 않습니다.
- 데이터 저장 구조, 테이블, 스키마, 내부 메시징·캐시 구성을 작성하지 않습니다.
- 인증·인가의 내부 구현, 토큰 검증 방식, 암호화·서명 방식, 키 관리 방법을 작성하지 않습니다.
- 보안 우회 방법, 공격 재현 절차, 내부 접근 제어 규칙, 취약점 대응 절차를 작성하지 않습니다.
- 비밀값, 자격 증명, 운영 명령, 장애 대응 절차, 내부 관측 정보를 작성하지 않습니다.

## 작성 원칙

- 대외 제품명은 `Hive Axyl`을 사용합니다.
- 기능은 외부 사용자의 입력, 공개 동작, 예상 결과 순서로 설명합니다.
- 내부 구현을 추측하거나 외부 동작의 근거로 공개하지 않습니다.
- 원본 자료에 내부 정보가 있어도 외부 사용자에게 필요한 결과만 남깁니다.
- 영어 문서와 `ko/` 문서의 기능 설명을 함께 갱신합니다.
- 내부 정보가 필요한 문서는 이 저장소에 만들지 않습니다.
